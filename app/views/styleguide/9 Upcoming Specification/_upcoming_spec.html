9.1

Set Vocabulary

Auszug, Vokabular and Kontexte tabs, according to the newst concepts, are to be simplified and merged within one single view as shown below.

The top container will accommodate items related to a vocabulary management. From here you can switch between vocabularies (by clicking on the dropdown) and selecting other items (not that active vocabularies are differentiated by slightly more contrasted font colour), you can enable/disable and view a currently selected vocabulary by clicking on the right-hand-side buttons.

The bottom container will accommodate words associated to currently selected vocabulary. User will be able to switch between All, Assigned and Most popular (old Auszug-like visualisation). Note variations of link meta links’ style when switching between the views:

  • in Vergebene view, terms with media assigned to them are active and others become inactive.
  • in Häufigste view, additional slider (same as the one we had before in Auszug view) appears to filter through the terms listed below.

9.2

Context page

Context view it the ones accessible while clicking on the View button in the Vocabulary view documented in Section 9.1. It comes with two tabs:

  • Vokabular: features a brief Vocabulary description (limited to 256 characters) and terms in that vocabulary with 3 display modes same as in the Set Vocabulary view.
  • Inhalte: in place of list of terms contained in currently viewed vocabulary, it features media associated to that vocabulary.

9.3

Filterbar in view of Filterset creation revamp.

The button linking to the filterset creation wizard is to be moved from search results page to the filterbar (so it will now be present in all views displaying thumbnails).

The 'Save as Filterset' button will open the modal window where user will be asked to enter filterset's desired name and either confirm or cancel the process.

The 'Rewind' button with the round arrow icon stands for resetting the filter set on the currently viewed batch of media—it will become active only after have applied at least one filter (as shown on the 2nd example).

When user is viewing a filterset that he previously created, and has editing permissions to, he will be able to save filterset settings he just altered (as shown on the 3rd example).

Users viewing a filterset, but not having permissions to edit it, will be presented with the option to duplicate currently set filterset settings as their own filterset—in other words—create a filterset on top of a filterset (as shown on the 4th example).
